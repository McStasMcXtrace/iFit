Requirements

Stand-alone / run-time release / no Matlab license required

When installing a 'stand-alone' release of iFit, there is simply NO dependency. The package contains everything required. You only need to have a running system. The stand-alone package is distributed for Linux, Windows and Mac OSX systems, for  x86_64 (64 bits) systems. See below to download the required stand-alone package.

The stand-alone version provides full iFit functionality at no cost. You can execute any script (list of commands) using iFit and Matlab functions. However, you can not define new functions.
Refer to the iFit page for more information on the features and limitations.

Matlab - source code release

The only requirement is to have Matlab installed. Any Matlab version from 7.4 (R2007a) will do, no need for any additional toolbox. This is a commercial product. Student versions can be bought for lower prices, as well as Academic licenses (e.g. for Universities). See below to download the required source package.

C compiler (optional)

In principle, the pre-compiled MeX files are shipped with the package (for e.g. looktxt), so that the software is ready to run. However, in the event of a re-compilation of MeX files, a C compiler is often shipped with Matlab (and installed automatically). The iFit/Loaders library may use it sporadically.

We prefer Linux systems, such as Ubuntu, when handling large data sets, and performing McStas optimizations, so that we fully use multi-core systems (e.g. though threads and MPI). You may use our ILL/CS Live DVD ready-to-run system with McStas pre-installed. Then add on top Matlab (requires a valid license).

Obtaining the package / Download

The package is currently available from the ifit.mccode.org web site:
  • source package for all platforms, requires a Matlab license to run [ zip, 14 Mb,  Changes ].
  • standalone packages for Linux, Windows and Mac OS X allow to execute iFit even without purchasing Matlab. See the installation instructions below.
    • Debian Linux 64 bits (incl. Ubuntu): R2010a MCR, then iFit debian package (compiled using Matlab R2010a on a Ubuntu 12.04 system)
    • Other Linux 64 bits: R2010a MCR, then iFit binary (compiled using Matlab R2010a on a Ubuntu 12.04 system)
    • Windows 64 bits: MCR, then iFit binary (compiled using Matlab R2010a on a W7 system)
    • Mac OSX 64 bits (Intel): R2011a MCR, then iFit binary (compiled using Matlab R2011a on a Snow Leopard system)
  • The development repository is available at McCode CVS.
Older releases can be obtained here.

Installation - stand-alone distribution

Install the R2010a Matlab Compiler Runtime (MCR) for your system if it has not be installed previously. This is not required if you only wish to upgrade iFit. Just click on OK all the way through.

On Linux Systems, we recommend to install the MCR in /opt/MATLAB (requires administrative privileges).

chmod u+x MCRINstaller.bin
sudo ./MCRInstaller.bin

Then install the iFit stand-alone package where-ever you wish. If you have not installer the MCR in /opt/MATLAB, you need to define the MCRROOT environment variable to its location prior to starting iFit :

export MCRROOT=/opt/MATLAB/MATLAB_Compiler_Runtime/v713/
ifit

On Linux systems, if you use the Debian package installer (.deb), nothing else is required. If you use a manual installation (.zip) we recommend that you extract the archive content in the directory /usr/local/ifit.

To launch iFit, click on the 'ifit' icon in the installation directory. You can also drag-n-drop files on the iFit icon to directly import data files and plot them.

Installation - source distribution

You just need to copy the iFit directory somewhere, say in /path/to.
  1. Extract first the iFit archive if needed.
  2. Just drag-drop the iFit directory in the target location that suits you.
For our users who love challenges and command lines, this may be achieved e.g. with
% unzip iFit.zip
% cp -r iFit /path/to
Start Matlab by clicking on its icon or typing the matlab command in a terminal.
% matlab
In order to use iFit, you need to import all its methods into Matlab each time you start it (this may be automated from the startup.m file). 
>> addpath(genpath('/path/to/iFit'))
This can also be done from the Matlab menu File/Set Path.../Add with subfolders... then enter the iFit location, and Save. That's all !

A list of fixed bugs and release history is recorded in the Changes page.

Note for Windows 64bits users: recompiling looktxt MeX files - source distribution

Windows 64 (XP, Vista, 7)
The Loaders sub library makes use of a MeX file 'looktxt.mexw64'. A pre-compiled version for Windows systems is shipped with the package but this binary may fail with some Matlab releases. In that case, it is required to install a compatible C compiler (currently Microsoft Visual C++). The required downloads are, in order, C++ Express then the SDK (for Matlab >= 2010).
Older Microsoft Visual C++ 2008 [ISO] can also be used together with the SDK for Windows 7 + NET 3.5 [amd64 ISO]). Just use the default installtion settings, then use
>> mex -setup							% select compiler
[1] Microsoft Windows VS2008
>> addpath(genpath('/location/to/iFit'))			% install iFit
>> delete ([ ifitpath 'Loaders/private/looktxt.mexw64' ])	% delete old looktxt MeX
>> iLoad force							% reload importer methods and recompile looktxt
An other solution, somewhat more complex to tune, is to use gcc (from e.g. MingW) and the gnumex tool. The Dev-Cpp compiler (which includes gcc) may also be used.

Then force the recompilation of looktxt with the procedure below.
Other systems (Windows 32, Linux, MacOSX)
All other platforms provide a valid C compiler
  • lcc for Windows 32bits (from Matlab)
  • gcc for Linux systems (comes with the gcc package)
  • gcc for MacOSX (need to install Xcode/developper tools, including gcc)
In order to force the recompilation of looktxt MeX, remove the iFit/Loaders/private/looktxt.mex* files, and launch
>> iLoad force
from the Matlab prompt. The C compiler to use may have to be specified with the command '>> mex -setup'.
Send me the new looktxt MeX file version in a tar.gz archive, so that I update/add it to the distribution.

What's next ?

You may test the package integrity by issuing the command:
>> ifittest
which will test, in a few minutes, all the examples given in this documentation and report failures. In principle all should go right.. Refer to the iFit Help in case of trouble.
You may start with the Quick Start page to learn what iFit can do for you.
