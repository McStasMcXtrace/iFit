Overview

The iFiles Utilities is a collection of Matlab functions designed for file access and operations.

  • iLoad:
  • InputStreamByteWrapper.java: Wrapper to the java.io.InputStream class for Matlab (Henrik Bengtsson). Used by geturl whenever available. If not available, an alternative slower method is used, but only works for text data (no binary, except compressed text).
  • geturl.m: enables to access an URL by opening a java.net.url connection. A local copy of any URL and compressed data is written locally (any previously existing local copy is renamed with an appended .old extension). Called by getfiles. This function is a simple wrapper to the Matlab urlwrite function when available (Matlab >= 6.5).

    • .
      local_filename = geturl(filename);            % create a local copy of a distant/compressed file
  • getfiles.m: checks file existence and resolves path names (including URL's and GZ/ZIP compressed data), and calls both geturl and uigetfiles, if necessary. The iStrings/iGrep Utilities are used to resolve regular expressions in file names.
      • filenames = getfiles(filenames);              % silent mode
      filenames = getfiles(filenames,'title'); % interactive mode (use Dialogs if necessary)
      filenames = getfiles([],'title');       % interactive mode (use uigetfiles and Dialogs if necessary)
  • uigetfiles.m: mutliple file selector (used just like uigetfile). Used by getfiles. See dedicated help for uigetfiles.
      • [filename, pathname]= uigetfiles;
The package main function are iLoad and getfiles.

Installation

This package is part of the ILL Matlab Library, and should have been installed by the main iLib/iInstall installation script.
For a better efficiency, the Java wrapper should be accessible by Matlab, and thus the path to the file InputStreamByteWrapper.class should be in the classpath.txt file of Matlab (type which 'classpath.txt' to locate it).
